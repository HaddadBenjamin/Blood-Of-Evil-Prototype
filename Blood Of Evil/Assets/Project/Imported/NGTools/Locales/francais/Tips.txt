Tips_Title=Astuces
Tips_Introduction=Toutes les choses à savoir qui vous permettront d'utiliser la toute puissance de la console, pour atteindre une efficacité maximale!\nQuoi de mieux que le meilleur cheval pour le plus grand chevalier?\nLa meilleure farine pour le meilleur boulanger?\nLa meilleure console pour le meilleur programmeur!
Tip_Console_Title=Une console pleine de surprises
Tip_Console_Content=Les différentes options <b>"Clear"</b>, <b>"Clear on Play"</b> et <b>"Error pause"</b> sont intrinsèquement liées à la console native d'Unity. Appuyez sur l'une reproduira l'exact comportement de la console native.\n\nFaites un <b>clic</b> sur le bouton <b>"Conf"</b> en haut à droite de la console pour mettre en surbrillance le fichier de configuration actuellement utilisé par la console.\nFaites un <b>double-clic</b> sur <b>"Conf"</b> pour sélectionner le fichier dans l'inspecteur.\n\nLe bouton <b>"Pref"</b> ouvre les préférences de la console. Depuis les préférences vous pouvez <b>changer la langue</b>, mais aussi créer d'autres fichiers de configuration, que vous pouvez sélectionner à tout moment. De cette façon, il est facile de <b>partager une configuration</b> entre plusieurs personnes, car tout réside dans un <b>fichier exportable</b>.\n\nChaque fichier de configuration contient la liste de tous les raccourcis, chaque raccourci est modifiable. Vous pouvez y accéder depuis la liste des fichiers de configuration dans les <b>préférences</b>, en cliquant sur le bouton à droite du fichier de configuration concerné.
Tip_Module_Title=Les modules d'abord!
Tip_Module_Content=La base fondamentale de la console est la classe <b>Module</b>. Depuis celle-ci vous pouvez interagir avec les autres <b>Module</b>, injecter votre code et modifier le comportement de la console.\n\nMainTModule est un <b>Module</b> dit actif, c'est-à-dire qu'il apparaît dans la barre des onglets, et il permet d'afficher tout simplement les logs.\n\nUn <b>Module</b> peut aussi être passif, ne pas s'afficher dans la barre des onglets, et modifier le comportement de la console.\nColorMarkersModule par exemple n'est pas visible, mais permet d'ajouter une couleur de fond aux logs selon des filtres définis pas l'utilisateur.\n\nVous pouvez créer votre propre implémentation en créant une classe héritant de <b>Module</b>.
Tip_Stream_Title=La toute puissance des fluxs
Tip_Stream_Content=Vous pouvez ajouter, changer le nom et supprimer autant de flux que vous le souhaitez.\nChaque flux fonctionne indépendamment, mais vous pouvez les rendre interactifs si vous activez l'option "Absorbe les logs entrants" représenté par un bouton Pac-MAN. Quand un log est absorbé, les autres fluxs qui ont activé cette option ne recevront pas le log.\nPrenez note que le flux principal n'est pas concerné par ces options et réceptionnera toujours tous les logs.\n\nChaque flux peut filtrer les logs par son type (Normal, avertissement, erreur, exception), ceux-ci sont indépendent des filtres annexes.\n\nFaites un <b>clic-droit</b> sur l'entête du flux pour modifier son nom ou le supprimer.\nFaites un <b>clic-milieu</b> pour directement supprimer le flux.\nChanger de flux en faisant appel aux raccourcis <b>Ctrl + Tab</b> ou <b>Ctrl + Shift + Tab</b> (Raccourcis par défaut).\n\nVous pouvez créer votre propre flux en créant une classe héritant de <b>StreamLog</b>.
Tip_Filter_Title=La subtilité des filtres
Tip_Filter_Content=Les filtres régulent l'acceptation des logs entrants. La règle est la suivante, le log sera refusé si au moins un filtre le rejette.\n\nChaque filtre peut ajouter une action dans le menu contextuel des logs, permettant des actions avec le log plus poussés et efficaces.\n\nFaites un <b>clic-droit</b> ou un <b>clic-milieu</b> sur l'entête du filtre pour le supprimer.\n\nVous pouvez créer votre propre filtre en créant une classe héritant de <b>ILogFilter</b>.
Tip_Log_Title=Faire de la gymnastique avec les logs
Tip_Log_Content=Les logs sont la représentation de ce que vous avez écrit ou le résultat d'une action d'Unity, aussi, vous pouvez ajouter un objet contextuel au log (Le 2e argument de la fonction Debug.Log/Warning/Error).\nQuand un objet contextuel est présent, une icône représentative du type de l'objet contextuel s'affiche sur la gauche du log.\n\nFaites un <b>clic-gauche</b> pour mettre en surbrillance l'objet dans la fenêtre Project ou Hierarchy.\nFaites un <b>double clic-gauche</b> pour aller à la ligne si il provient d'un fichier source, ou pour ouvrir le fichier pour les autres types d'objet.\nFaites un <b>clic-milieu</b> pour afficher/cacher le tracé d'appels si il y en a un.\nFaites un <b>clic-droite</b> pour ouvrir le menu contextuel.\n\nLe fichier de configuration possède 2 paramètres améliorant l'interaction avec les logs. Voir  Log->"Force Focus On Modifier" et Log->"Select Object On Modifier".\n"Force Focus On Modifier" permet d'aller à la ligne ou d'ouvrir l'objet en un <b>clic</b> + <b>Modifier</b>.\n"Select Object On Modifier" permet de sélectionner l'objet contextuel en un <b>clic</b> + <b>Modifier</b>.\n\nIl est possible de sélectionner plusieurs logs grâce à la multi-sélection, maintenez <b>Ctrl</b> appuyé et cliquez sur des logs.\n\nAppuyez sur <b>Ctrl + C</b> (Windows) pour copier la première ligne de chaque log sélectionné.\nFaites un <b>double Ctrl + C</b> ou <b>Ctrl + X</b> (Windows) pour copier le contenu entier de chaque log sélectionné.\n\nQuand il n'y a qu'un seul log sélectionné, son contenu s'affiche en bas de la console séparé par une barre de séparation.\nFaites un <b>double-clic</b> ou un <b>clic-droit</b> sur la barre de séparation pour l'ajuster automatiquement à la bonne taille pour visualiser le contenu en entier.
Tip_Translation_Title=La traduction dans tous ses états
Tip_Translation_Content=Quasiment tout est traduisible!\nSi vous voulez ajouter une nouvelle langue, il vous est libre de créer un répertoire dans le dossier <b>"NGConsole/Locales"</b>.\nEnsuite faites réimporter n'importe quel fichier de code pour forcer la mise à jour des langages.\nEnfin ouvrez la fenêtre des préférences et votre nouvelle langue apparaîtra dans la liste.\n\nTous les fichiers de traduction doivent se terminer par l'extension <b>".txt"</b>.\nLe format est simple, chaque clé est séparé par une nouvelle ligne, et chaque ligne doit comporter une clé suivi du caractère de séparation "=" et terminer par la traduction.\nVos traductions peuvent contenir des balises de marquages.\nPour insérer un saut de ligne, ajouter un "\" suivi d'un "n".\n\nExemple:\nClé1=Lorem ipsum dolor sit amet, consectetur adipiscing elit.\nClé2=Nulla convallis, diam sit amet eleifend laoreet.
Tip_Tip_Title=Astuce-ception
Tip_Tip_Content=Vous pouvez ajouter vos propres astuces dans cette fenêtre en ajoutant une clé grâce à <b>TipsWindow.AddTip</b>.\nLa clé fournie doit être accompagnée de 2 clés de traduction qui sont "Tip_{clé}_Title" et "Tip_{clé}_Content".
